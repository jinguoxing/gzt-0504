# 待我处理页面｜前端交互测试用例

## 1. 路由测试

### TC-001：打开待我处理页面

操作：访问 `/tasks/reviews?assignee=me`

预期：

```text
页面正常打开
标题为“待我处理”
左侧导航高亮“任务”
未进入任务执行页
```

### TC-002：路由顺序测试

操作：检查 routes 配置。

预期：

```text
/tasks/reviews 在 /tasks/:taskId 前面
```

## 2. Tab 测试

### TC-003：切换待确认

操作：点击“待确认”。

预期：

```text
列表只显示待确认事项
URL 包含 type=CONFIRM
```

### TC-004：切换异常

操作：点击“异常”。

预期：

```text
列表只显示异常事项
URL 包含 type=EXCEPTION
```

## 3. 搜索测试

### TC-005：搜索订单金额

操作：输入“订单金额”。

预期：

```text
只显示“字段语义确认：订单金额”相关事项
```

## 4. 筛选测试

### TC-006：筛选高优先级

操作：优先级选择“高”。

预期：

```text
只显示高优先级事项
```

### TC-007：筛选来源任务

操作：来源任务选择“元数据自动采集任务”。

预期：

```text
只显示该来源任务下的异常事项
```

## 5. 批量操作测试

### TC-008：选择两条事项

操作：勾选两条事项。

预期：

```text
显示“已选择 2 个事项”
显示批量确认 / 批量转派 / 批量忽略
```

### TC-009：清空选择

操作：点击批量操作条关闭按钮。

预期：

```text
已选中事项清空
批量操作条隐藏
```

## 6. 行级操作测试

### TC-010：进入确认

操作：点击“进入确认”。

预期：

```text
跳转到 /tasks/:taskId/review/:reviewId
```

### TC-011：查看异常

操作：点击“查看异常”。

预期：

```text
跳转到 /tasks/:taskId/review/:reviewId?mode=exception
```

### TC-012：继续执行

操作：点击“继续执行”。

预期：

```text
跳转到 /tasks/:taskId?stageId=:stageId&resume=true
```

## 7. 右侧栏测试

### TC-013：快捷视图高优先级

操作：点击“高优先级事项”。

预期：

```text
列表筛选 priority=HIGH
URL 同步
```

### TC-014：快捷视图今日到期

操作：点击“今日到期”。

预期：

```text
列表筛选 deadline=today
URL 同步
```

## 8. 空状态测试

### TC-015：筛选无结果

操作：设置一个无结果筛选。

预期：

```text
显示“没有符合条件的事项”
显示“重置筛选”按钮
```

## 9. 视觉测试

### TC-016：全局外壳一致

检查：

```text
Logo 是 Semovix
左侧导航顺序正确
顶部栏结构正确
用户是李桐
当前高亮任务
```
