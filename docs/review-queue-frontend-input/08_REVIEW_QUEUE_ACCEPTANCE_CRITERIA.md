# 待我处理页面｜验收标准

## 1. 路由验收

- [ ] 存在 `/tasks/reviews` 路由。
- [ ] `/tasks/reviews` 位于 `/tasks/:taskId` 动态路由之前。
- [ ] 访问 `/tasks/reviews?assignee=me` 能正确打开待我处理页面。
- [ ] 页面左侧导航高亮“任务”。
- [ ] 从任务中心点击“待我处理 查看全部”进入 `/tasks/reviews?assignee=me`。

## 2. 页面对象验收

- [ ] 页面展示对象是“事项”，不是“任务”。
- [ ] 表格行名称类似“字段语义确认：订单金额”，而不是只展示“供应链语义治理闭环任务”。
- [ ] 每条事项包含来源任务和所属阶段。
- [ ] 每条事项都有明确操作按钮。

## 3. UI 结构验收

- [ ] 页面包含面包屑“任务 / 待我处理”。
- [ ] 页面标题为“待我处理”。
- [ ] 顶部有类型 Tab：全部 / 待确认 / 待审核 / 异常 / 继续执行。
- [ ] 有 5 个指标卡：全部事项 / 待确认 / 待审核 / 异常处理 / 继续执行。
- [ ] 有搜索与筛选工具栏。
- [ ] 有批量选择提示条。
- [ ] 有事项表格。
- [ ] 有分页。
- [ ] 有右侧处理概览 / 快捷视图 / 说明提示。

## 4. 交互验收

- [ ] 点击 Tab 能筛选事项，并同步 URL query。
- [ ] 搜索事项名称能过滤列表。
- [ ] 状态、类型、来源任务、发起人、优先级、截止时间可组合筛选。
- [ ] 重置筛选能恢复默认状态。
- [ ] 排序能按优先级、截止时间、发起时间变化。
- [ ] 勾选事项后显示批量操作条。
- [ ] 批量确认、批量转派、批量忽略按钮存在。
- [ ] 点击事项名称进入事项详情页或打开详情抽屉。
- [ ] 点击来源任务进入任务详情页。
- [ ] 点击进入确认 / 去审核 / 查看异常 / 继续执行能跳转到正确页面。

## 5. 视觉验收

- [ ] 与 Semovix 全局外壳一致。
- [ ] Logo、顶部栏、左侧导航、用户信息未被重做。
- [ ] 左侧导航顺序正确。
- [ ] 当前页面高亮“任务”。
- [ ] 颜色只使用蓝 / 绿 / 橙 / 红 / 灰。
- [ ] 表格文字清晰，行高适中。
- [ ] 右侧栏不抢主表格视觉。
- [ ] 没有营销海报风、BI 大屏风、传统聊天气泡。

## 6. 数据验收

- [ ] 有至少 8 条 mock 事项。
- [ ] 覆盖待确认、待审核、异常、继续执行四种类型。
- [ ] 覆盖高 / 中 / 低优先级。
- [ ] 覆盖今天到期、明天到期、尽快处理。
- [ ] 统计卡数量与列表数据一致或明确来自汇总字段。

## 7. 状态验收

- [ ] 空状态有明确文案和返回入口。
- [ ] 加载态使用骨架屏。
- [ ] 错误态有“重新加载”按钮。
- [ ] 筛选无结果时有“重置筛选”入口。

## 8. 不得出现的问题

- [ ] 不得把待我处理页做成全部任务页。
- [ ] 不得展示完整任务库字段作为主对象。
- [ ] 不得缺失 `/tasks/reviews` 路由。
- [ ] 不得让 `/tasks/reviews` 被 `/tasks/:taskId` 误匹配。
- [ ] 不得改变全局产品外壳。
