# 待我处理页面规格说明

## 1. 页面基本信息

| 项 | 内容 |
|---|---|
| 页面名称 | 待我处理 |
| 页面路由 | `/tasks/reviews` |
| 页面对象 | ReviewQueueItem / ActionItem / 待处理事项 |
| 页面归属 | 任务模块 |
| 左侧导航高亮 | 任务 |
| 默认视图 | 全部 |
| 核心目标 | 展示当前用户需要处理的事项，并支持快速处理 |

## 2. 页面定位

待我处理页面是用户的事项处理队列，不是完整任务库。

页面需要回答：

```text
我现在有哪些事项必须处理？
哪些事项优先级最高？
哪些事项即将到期？
哪些事项异常？
每个事项来自哪个任务和阶段？
我应该点击哪个按钮继续？
```

## 3. 与其他页面的区别

| 页面 | 展示对象 | 主要用途 |
|---|---|---|
| `/tasks` | 摘要、行动入口 | 看今天要做什么 |
| `/tasks/all` | Task | 搜索、筛选、管理全部任务 |
| `/tasks/reviews` | ActionItem / ReviewItem | 处理我的待办事项 |
| `/tasks/:taskId` | 单个任务 | 查看任务执行过程 |
| `/tasks/:taskId/review/:reviewId` | 单个事项 | 处理具体确认、审核、异常 |

## 4. 页面结构

```text

```


右侧辅助栏：

```text
1. 处理概览
2. 快捷视图
3. 说明 / 提示
```

## 7. 指标概览卡

| 指标 | 数值 | 说明 |
|---|---:|---|
| 全部事项 | 17 | 当前用户所有待处理事项 |
| 待确认 | 6 | 字段、对象、交付物等确认事项 |
| 待审核 | 4 | 业务审核、对象审核、映射审核 |
| 异常处理 | 3 | 数据源异常、Schema 失败、权限异常 |
| 继续执行 | 4 | 被暂停、中断、等待继续的任务 |
