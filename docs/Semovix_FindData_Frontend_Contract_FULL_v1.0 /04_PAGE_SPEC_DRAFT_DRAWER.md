# 找数问数草稿/确认态页面规格 v1.0

## 1. 重要结论

找数问数默认不需要治理类“任务草稿抽屉”。

用户问：

```text
上个月采购金额是多少？
```

应直接进入找数问数执行页，不应出现：

```text
任务草稿
8 个阶段
创建并开始执行
治理类任务计划
```

---

## 2. 什么时候需要确认态？

仅在以下情况下出现轻量确认态。

| 场景 | 原因 | 处理方式 |
|---|---|---|
| 口径不明确 | 可能有多个指标解释 | 展示口径确认卡 |
| 时间范围不明确 | “最近”“这个月”可能有歧义 | 展示时间确认 |
| 指标候选多个 | 销售额可能有含税/不含税/净额 | 展示指标候选 |
| 查询范围过大 | 可能扫描大量数据 | 展示范围确认 |
| 敏感数据 | 需要确认权限或脱敏 | 展示权限/风险确认 |
| 生成分析报告 | 从问数升级为分析交付 | 展示分析任务确认 |
| 发起治理任务 | 从问数进入治理闭环 | 跳转治理类草稿/任务页 |

---

## 3. 轻量确认态不是右侧任务草稿

找数问数确认态建议使用中间结果卡或右侧轻抽屉。

### 推荐名称

```text
口径确认
查询确认
分析确认
```

不要叫：

```text
任务草稿
任务计划
执行阶段
```

---

## 4. 口径确认卡结构

### 示例问题

```text
这个月销售怎么样？
```

### 页面展示

```text
Xino · 刚刚
这个问题涉及多个可能口径，请确认你想查看的指标。

[ClarificationCard]
你想看的是？
- 销售额
- 订单量
- 毛利
- 客单价

时间口径：
- 自然月
- 财务月

订单范围：
- 已支付
- 已发货
- 全部订单

Xino 推荐：
默认使用“已支付订单销售额 + 自然月口径”。

[按推荐口径查询] [手动选择] [取消]
```

## 5. 查询确认卡结构

用于查询范围过大或涉及敏感数据。

```text
本次查询预计扫描 1,840,320 行数据。
数据源：supply_chain_prod
来源表：dwd_scm_purchase_order_line
预计耗时：约 8 秒
权限状态：可查询

[继续查询] [限制范围] [取消]
```

## 6. 从问数升级为分析任务

用户输入：

```text
帮我分析为什么本月采购金额上涨，并生成报告。
```

这已经不是简单问数，属于轻量分析任务。

此时可以出现分析确认态：

```text
分析目标：采购金额上涨原因分析
分析步骤：
1. 查询核心指标
2. 做环比分析
3. 按供应商/品类拆解
4. 识别贡献因素
5. 生成分析报告

[开始分析] [调整范围] [保存草稿]
```

但仍然不要复用治理类 8 阶段任务草稿。

## 8. 验收标准

```text
简单问数不得出现草稿确认。
口径不清时必须先确认，不得乱答。
确认态必须轻量，不得做成治理任务草稿。
确认后进入找数问数执行页。
只有用户明确发起治理/修复任务时，才进入治理类任务页。
```
