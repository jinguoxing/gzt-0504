# 找数问数交互规则 v1.0

## 1. 首页提交问题

```text
用户输入问题
点击发送
Xino 识别意图
若 intent=data_qa，创建会话并跳转执行页
```

执行页不得再展示当前问题输入区。

---

## 2. 执行页加载

加载顺序：

```text
1. 显示页面标题。
2. 显示用户提问条。
3. 显示 Xino 正在查询状态。
4. 查询完成后显示 Xino 文本说明。
5. 渲染结果模板组。
6. 加载右侧答案依据。
```

---

## 3. Xino 回复规则

Xino 每次回复必须包含：

```text
文本说明
结果模板组
推荐追问 / 后续动作
```

禁止只给图表或只给表格。

---

## 4. 继续追问

用户在底部输入框继续追问。

示例：

```text
为什么本月采购金额上涨？
```

前端行为：

```text
1. 追加用户提问条。
2. 显示 Xino 分析中状态。
3. 追加 Xino 文本说明。
4. 追加原因解释、贡献度分析、后续动作。
5. 更新右侧追问上下文。
```

---

## 5. 推荐追问点击

用户点击推荐 chip：

```text
为什么上涨？
按供应商拆解
看最近 30 天趋势
导出明细
```

行为等同于用户输入对应问题。

---

## 6. 查看来源

用户点击：

```text
查看完整来源
```

右侧栏切换为 `source_detail`，展示数据来源详情。

---

## 7. 查看 SQL

用户点击：

```text
查看 SQL
```

行为：

```text
右侧栏展开“查询计划 / SQL”
滚动到 SQL 区域
高亮 SQL 卡片
```

---

## 8. 查看明细

用户点击：

```text
查看全部
```

可选两种实现：

```text
方案 A：中间展开完整表格
方案 B：右侧或弹层打开明细表
```

第一期建议使用方案 B，避免中间区过长。

---

## 9. 导出明细

用户点击：

```text
导出明细
```

状态：

```text
exporting → export_success / export_failed
```

成功后展示交付物：

```text
采购订单明细.xlsx
```

---

## 10. 口径不清

若问题不清，禁止直接查询。

展示：

```text
ClarificationCard
```

用户确认后再查询。

---

## 11. 权限不足

若数据源或字段不可查询，展示：

```text
FallbackResultCard
原因说明
影响范围
建议动作：申请权限 / 更换口径 / 更换数据源
```

---

## 12. 右侧栏折叠规则

默认：

```text
当前口径：展开
数据依据：展开
查询计划 / SQL：折叠
追问上下文：折叠
```

用户点击可展开。

---

## 13. 从问数进入治理任务

当用户点击：

```text
发起治理任务
生成修复任务
```

前端行为：

```text
1. 生成治理任务草稿。
2. 带入当前问数结果、SQL、来源表、问题上下文。
3. 跳转治理任务流程。
```

---

## 14. 禁止行为

```text
不得把找数问数做成治理类任务计划。
不得在执行页顶部再放问题输入框。
不得使用聊天气泡。
不得首屏展开大表格。
不得把 SQL 默认大面积展开。
```
