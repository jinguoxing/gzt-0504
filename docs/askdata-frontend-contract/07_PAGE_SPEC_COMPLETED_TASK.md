# 找数问数完成态页面规格 v1.0

## 1. 页面定位

找数问数通常不是长任务，但以下情况会进入完成态：

```text
用户保存本次问答结果
用户导出明细
用户生成分析报告
用户共享给团队
用户将问数结果转成治理/修复任务
```

完成态用于归档和复用本次问数结果。

---

## 2. 进入条件

| 用户动作 | 完成态内容 |
|---|---|
| 保存结果 | 保存本次问答和答案依据 |
| 导出明细 | 生成 Excel / CSV 文件 |
| 生成分析报告 | 生成 PDF / Markdown 报告 |
| 分享 | 生成可分享链接 |
| 发起治理任务 | 跳转治理任务，同时保留来源 |

---

## 3. 页面结构

```text
左侧导航
顶部全局栏
中间完成摘要
右侧结果归档栏
```

---

## 4. 中间完成摘要

标题：

```text
问数结果已保存
```

摘要：

```text
问题：上个月采购金额是多少？
答案：¥ 12,486,320
环比：+8.6%
同比：+12.3%
统计周期：2026-04-01 至 2026-04-30
口径：已审核采购订单含税金额
```

---

## 5. 追问链路

```text
1. 上个月采购金额是多少？
2. 为什么本月采购金额上涨？
```

每条追问可以展开查看结果卡片摘要。

---

## 6. 交付物

如果用户导出或生成报告，展示：

```text
采购金额问数结果.xlsx
采购金额变化原因分析.pdf
查询 SQL.txt
答案依据.json
```

每个文件操作：

```text
预览
下载
分享
复用
```

---

## 9. 从完成态进入治理任务

当用户点击：

```text
发起采购金额异常治理任务
```

前端应：

```text
1. 创建治理任务草稿。
2. 将本次问数结果作为上下文资源。
3. 跳转 /tasks/:taskId 或治理草稿确认页。
```

---

## 10. 验收标准

```text
完成态必须保留答案、口径、来源和追问链路。
交付物必须可下载或预览。
用户能从完成态继续追问或发起治理任务。
```
