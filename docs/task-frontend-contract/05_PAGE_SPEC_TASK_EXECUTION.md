# 05｜Page Spec｜任务执行页 / 执行中主态

## 1. 页面目标

任务执行页是用户确认草稿后进入的正式任务执行页面。页面核心不是聊天，而是：

```text
当前阶段对话 + 当前阶段结果 + 右侧任务计划 + 人机确认入口
```

---

## 2. 左侧导航高亮

```text
任务
```

## 6. 任务流式对话

禁止传统聊天气泡，采用任务事件流。

用户消息：

```text
李桐 09:45
请继续推进字段语义理解，并重点识别冲突字段。
```

Xino 消息：

```text
Xino 09:46 执行中
正在进行字段语义理解与冲突检测，已解析字段 4,920 个，已理解 3,812 个。
```

执行进度：

```text
字段解析 100% / 语义理解 78% / 冲突检测 62%
```

## 7. 当前阶段结果卡片

标题：

```text
字段语义理解结果
```

结构：

```text
1. Xino 结论摘要
2. 指标组
3. 结果表格
4. 推荐动作
```

结论摘要：

```text
本阶段已完成大部分字段的语义理解，高置信字段可批量确认，冲突字段建议优先处理。
```

指标：

```text
自动通过字段：3,812
待确认字段：326
冲突字段：41
异常字段：37
建议忽略字段：704
```

表格列：

```text
字段名 / 推断语义 / 置信度 / 来源 / 操作
```

示例：

| 字段名 | 推断语义 | 置信度 | 来源 | 操作 |
|---|---|---:|---|---|
| po_status | 采购订单状态 | 0.97 | 业务词典 + 规则推断 | 确认 / 修改 / 忽略 |
| supplier_code | 供应商编码 | 0.96 | 业务词典 + 相似匹配 | 确认 / 修改 / 忽略 |
| wh_id | 仓库ID | 0.93 | 业务词典 + 规则推断 | 确认 / 修改 / 忽略 |
| create_dt | 创建日期 | 0.89 | 规则推断 | 确认 / 修改 / 忽略 |

底部链接：

```text
查看全部 4,920 个字段
```

## 11. 禁止

- 不要传统聊天气泡。
- 不要展开所有历史阶段。
- 不要把任务计划和任务详情混在一起。
- 不要让右侧栏信息过载。
- 不要只用纯文本展示结果。
