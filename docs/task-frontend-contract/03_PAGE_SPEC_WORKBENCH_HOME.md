# 03｜Page Spec｜AI 工作台首页

## 1. 页面目标

AI 工作台首页是用户发起任务的轻量主入口。用户不需要先填复杂表单，而是直接告诉 Xino 想完成什么。

核心目标：

```text
让用户以自然语言表达目标 → Xino 理解目标 → 进入任务草稿确认态
```

---

## 2. 左侧导航高亮

```text
AI 工作台
```

---

## 3. 页面布局

```text
AppShell
  LeftNav
  TopBar
  MainContent
```

首页默认不显示右侧栏。

---

## 4. 主内容区

### 4.1 主标题

```text
今天想让 Xino 帮你完成什么？
```

### 4.2 副标题

```text
把目标告诉我，Xino 会理解你的需求，并规划可执行的任务路径。
```

### 4.3 输入框

Placeholder：

```text
输入任务目标，或 @ 数据源 / 文件 / 对象 / 知识库
```

输入框底部弱入口：

```text
附件
@ 资源
语音
历史
```

主按钮：

```text
发送给 Xino
```

禁止使用：

```text
生成任务草稿
```

---

## 5. 首页辅助入口

输入框下方只显示折叠入口，不默认展开。

```text
常用任务 4 个  >
最近继续 3 个  >
常用资源 5 个  >
```

底部弱提示：

```text
Xino 会根据你的目标，自动识别任务类型、推荐数据源、规划执行步骤。
```

---

## 6. 行为

### 6.1 发送任务目标

用户点击“发送给 Xino”：

1. 输入非空校验。
2. 显示 Xino 正在解析。
3. 生成 draftId。
4. 当前页右侧滑出 DraftDrawer。

### 6.2 @ 资源

`@` 入口可选择：

- 数据源
- 文件
- 对象
- 知识库

### 6.3 折叠入口

点击折叠入口可展开轻量列表，但不得变成复杂门户页面。

---

## 7. 空态与错误态

| 场景 | UI |
|---|---|
| 输入为空 | 输入框下方显示“请输入任务目标” |
| 解析失败 | Xino 消息块提示失败，可重试 |
| 资源选择失败 | toast 提示，不影响主输入 |

---

## 8. 禁止

- 不要传统聊天气泡。
- 不要默认展示大号上传/数据源/模板卡片。
- 不要让首页像任务中心。
- 不要复杂表单。
- 不要伪中文、乱码、错别字。
