# 00｜Frontend Overview

## 1. 项目定位

Semovix 是一个面向企业数据语义治理、业务对象建模、字段语义理解、数据资产治理与智能任务执行的 AI 工作台。前端需要呈现一条完整工作流：

```text
用户提出目标
→ Xino 理解目标
→ 生成任务草稿
→ 用户确认/调整
→ 创建任务并执行
→ 分阶段展示结果
→ 人工确认/审核
→ 生成交付物
→ 进入下一轮治理或复用成果
```

本合同包覆盖以下核心页面和状态：

| 页面/状态 | 路由建议 | 作用 |
|---|---|---|
| AI 工作台首页 | `/workbench` | 用户通过自然语言发起任务 |
| 草稿抽屉态 | `/workbench?draftId=:id` | 任务正式创建前确认目标、范围、数据源、交付物 |
| 任务中心 | `/tasks` | 查看运行中的任务、待我处理、最近交付物和任务看板 |
| 全部任务列表 | `/tasks/all` | 搜索、筛选、排序、批量管理全部任务 |
| 任务执行页 | `/tasks/:taskId` | 当前阶段执行、阶段结果、任务计划、风险与交付物 |
| 字段/对象确认页 | `/tasks/:taskId/review/:reviewId` | 人工确认、冲突处理、审核细节 |
| 完成态/交付物页 | `/tasks/:taskId/completed` | 任务总结、交付物、下一步推荐 |

> 目录中保持用户指定的 00-14 文件结构。任务中心与全部任务列表作为新补充场景，已写入总览、流程、路由、组件、交互与实现提示中。

## 3. 全局视觉原则

### 3.1 视觉气质

```text
现代企业级 SaaS
白色 / 浅灰背景
Semovix 蓝为主色
专业、克制、稳定、清晰
圆角卡片
轻阴影或无阴影
高可读性中文字体
线性图标
```

不要做成：

```text
营销海报
BI 大屏
花哨科技风
传统聊天软件
复杂后台表单堆叠
```

### 3.2 色彩原则

颜色只表达状态和优先级，不表达热闹。

| 类型 | 用途 |
|---|---|
| 蓝色 | 品牌、当前选中、执行中、主按钮、进度、链接 |
| 绿色 | 已完成、成功、通过 |
| 橙色 | 待处理、待确认、待审核、中优先级 |
| 红色 | 异常、失败、高风险、高优先级 |
| 灰色 | 背景、边框、普通信息、未开始、已归档 |

禁止每个功能入口使用不同高饱和背景色。

## 4. 关键产品对象

| 对象 | 说明 |
|---|---|
| Task | 正式任务，包含目标、类型、阶段、进度、状态、负责人、交付物 |
| Draft | 任务草稿，尚未正式执行，可通过对话或配置调整 |
| Stage | 任务阶段，包含状态、进度、产出、风险和下一步动作 |
| ReviewItem | 待处理/待审核事项，例如字段确认、冲突处理、对象审核 |
| DataSource | 数据源，例如 MySQL 的 `supply_chain_prod` |
| ContextResource | 上下文资源，例如 PDF、XLSX、DOCX |
| Deliverable | 交付物，例如报告、清单、ZIP、图谱文件 |
| Activity | 用户或系统活动记录 |
| SemanticAsset | 语义资产，例如业务对象、字段语义、关系图谱 |
