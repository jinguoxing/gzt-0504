# 04｜Page Spec｜AI 工作台草稿抽屉态

## 1. 页面目标

用户在首页输入任务目标后，不跳转独立页面，而是在当前 AI 工作台中：

```text
中间保留任务流式对话
右侧滑出任务草稿抽屉
```

此时任务尚未正式执行。

---

## 2. 左侧导航高亮

```text
AI 工作台
```

---

## 3. 页面布局

```text
AppShell
  LeftNav
  TopBar
  MainContent: ConversationStream
  RightDrawer: DraftDrawer
```

右侧草稿抽屉宽度：`520px - 560px`。

---

## 4. 中间对话区

### 4.1 标题

```text
AI 工作台
```

### 4.2 副标题

```text
与 Xino 对话，确认任务目标、范围与执行方案。
```

### 4.3 对话呈现规则

- 不使用传统聊天气泡。
- 不使用左右对话布局。
- 使用纵向任务流消息块。
- 每条消息包含角色、时间、正文。

### 4.4 用户消息样例

```text
你 10:30
请对供应链数据库进行语义治理，扫描 Schema、识别业务相关表、理解字段语义、生成业务对象与交付物。
```

### 4.5 Xino 消息样例

```text
Xino 10:31
我已理解你的目标，并整理出一个任务草稿。右侧是当前任务配置，你可以直接确认，也可以继续告诉我需要调整的内容。
```

建议调整 chip：

```text
只扫描采购相关 Schema
增加 Excel 明细文件
增加字段质量报告
调整并行度为 10
```

---

## 5. 对话调整与草稿联动

用户追加：

```text
只扫描采购相关 Schema，不要扫描全库。
```

Xino 变更摘要卡：

```text
已更新任务草稿
扫描范围：全库 → ods_scm, dwd_scm
预计扫描表数量：286 → 142
预计耗时：2.5 小时 → 1.4 小时
风险等级：中 → 低
```

右侧草稿抽屉对应配置项高亮，并显示：

```text
刚刚通过对话更新
```

---

## 6. 右侧 DraftDrawer

### 6.1 顶部

```text
任务草稿 · 待确认
尚未开始执行
```

右上角：关闭按钮。

### 6.2 任务理解

```text
任务名称：供应链语义治理闭环任务
目标摘要：对供应链数据库进行语义治理，扫描 Schema，识别业务相关表，理解字段语义，生成业务对象与交付物。
任务类型：语义治理 / Schema 扫描 / 业务对象生成 / 交付物生成
```

### 6.3 关键配置

```text
数据源：supply_chain_prod
MySQL · 10.10.10.25:3306
状态：已连接

扫描范围：ods_scm, dwd_scm
共 2 个 Schema
标记：刚刚通过对话更新

上下文资源：3 个文件
PDF 1 个，XLSX 1 个，DOCX 1 个

交付物：6 项
对象模型、字典、血缘、质量报告等
```

### 6.4 推荐执行方案

```text
供应链语义治理方案
推荐标签
基于供应链领域最佳实践的治理流程与规则配置方案
```

### 6.5 风险提醒

```text
预计扫描约 142 张表、4,920 个字段
部分字段命名不规范，可能需要人工确认
当前扫描范围已限制为采购相关 Schema
```

### 6.6 执行阶段摘要

```text
预计 8 个阶段
约 1.4 小时
```

### 6.7 高级配置摘要

默认折叠：

```text
高级配置：范围、采样、并行度、输出、权限、异常策略
```

展开后包括：

- Schema / 表范围
- 字段采样策略
- 执行并行度
- 执行时间窗口
- 输出格式
- 权限与可见范围
- 自动确认阈值
- 异常中断与重试策略

---

## 7. 底部操作

固定在抽屉底部：

```text
保存草稿
高级配置
创建并开始执行
```

主按钮：`创建并开始执行`。

---

## 8. 禁止

- 不要让用户误以为任务已经执行。
- 不要把草稿抽屉做成独立页面。
- 不要让高级配置默认铺满页面。
- 不要传统聊天气泡。
